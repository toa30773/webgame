# 戦将（SenSho）

## プロジェクト概要

このリポジトリはスマートフォン専用Webアプリ「戦将（SenSho）」の開発プロジェクトです。

Claude Codeは、プロジェクト内の「仕様書」フォルダにあるすべてのMarkdownファイルを読み、仕様を理解した上で開発を行ってください。

対象ファイル

- 仕様書/01.md
- 仕様書/02.md
- 仕様書/03.md
- 仕様書/04.md
- 仕様書/05.md

今後追加される仕様書も必ず参照してください。

---

# 実装ルール

- 仕様書を唯一の正式な仕様として扱うこと。
- 実装中に仕様とコードが矛盾した場合は仕様書を優先すること。
- 不足している仕様は勝手に決めず、必ず質問すること。
- 一度に全てを実装せず、Phaseごとに実装すること。
- 実装前に設計方針を説明すること。
- TypeScript Strict Mode を前提に実装すること。
- any型は使用しないこと。
- 保守性・可読性・拡張性を最優先すること。
- ゲームロジックとUIを分離すること。
- コンポーネントは責務ごとに分割すること。
- 大きな機能は小さなコミット単位で実装すること。
- パフォーマンスを意識し、スマートフォンで60FPSを目標とすること。

---

# 開発方針

最優先は「遊べるゲーム」を作ることです。

デザインやエフェクトよりもゲームシステムを優先してください。

まずはプロトタイプを完成させ、実際にプレイできる状態を目標としてください。

クラロワを参考にしていますが、コピーは避け、「将軍を直接操作して軍を指揮するRTS」という独自性を重視してください。

仕様書にない内容を追加する場合は、実装前に提案してください。
